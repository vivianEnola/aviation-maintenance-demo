# 数据标注规范

预标注只用于减少操作量，不能直接作为最终真值。每张预标注图片都必须人工确认并在审核清单中改为 `approved`。

## 通用规则

- 保留原始 RGB 图片，不在标注软件中覆盖原图。
- 同一原图、同一机场或同一次气象事件的裁片必须属于同一数据切分组。
- 标签名称只能使用本项目规定的英文名称，不能改大小写或添加中文别名。
- 不确定时标记为 `needs_review`，不要为了凑数量强行标注。
- 没有目标的清晰图片是有效负样本，应保留空标签文件。

## 场景分类

无需标注软件，按“图片最适合交给哪个下游模型”放入四个类别文件夹：

1. `runway_detail`：跑道表面细节足以分析；
2. `airport_overview`：能识别机场整体和周边环境，但跑道表面细节不足；
3. `cloud_scene`：主要内容是云层或涡旋状天气系统；
4. `other`：不满足以上条件。

## 云检测（LabelImg，YOLO 格式）

类别文件：`labeling/cloud_classes.txt`。

- `cumulonimbus`：框住连续、视觉特征一致的积雨云主体。
- `stratocumulus`：框住连续的层积云云团，不把整个天空机械地框成一个目标。
- `typhoon_vortex`：框住完整可见的白色涡旋结构；标签表示视觉形态，不表示官方确认的台风。
- 无法判断云型的云图作为负样本或移入待复核，不强制标注。

## 机场周边检测（LabelImg，YOLO 格式）

类别文件：`labeling/airport_classes.txt`。

- `airport`：框住完整可见的机场主体区域。
- `large_building_cluster`：框住视觉上连续的大型建筑或建筑群，不逐栋标普通小建筑。
- `construction_area`：只标具有裸土、施工机械、未完成结构等明显施工特征的区域。
- 不标“高楼”或“违建”，模型没有高度和审批信息。

## 跑道状态分割（Labelme，Polygon）

类别文件：`labeling/runway_classes.txt`。

- `runway`：沿可见跑道铺装边界绘制多边形；被积雪或积水覆盖的跑道仍需标出跑道范围。
- `snow`：只标跑道上可见的积雪区域。
- `standing_water`：只标具有可靠视觉依据的积水区域；阴影、普通湿润面和深色修补区域不作为确定积水。
- 跑道多边形可以与积雪、积水多边形重叠。
- Labelme JSON 与图片同名保存，使用转换脚本生成 YOLO segmentation 标签。

## 预标注审核清单

逐张执行：

1. 删除误检；
2. 补充漏检；
3. 修正类别；
4. 收紧或修正边界框/多边形；
5. 检查图片中是否仍有未标目标；
6. 将审核状态从 `needs_review` 改为 `approved`。
